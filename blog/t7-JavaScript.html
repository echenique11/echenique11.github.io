Ruby Hashes vs. JavaScript Objects

3 - 22 - 2015

At first glance it looks like Ruby Hashes and JavaScript Objects are similar, but to me the only thing that they have in common is the way the store data, both uses a list of key : values pair, where giving a key you'll get it values. For example : In JavaScript you create an object 

				var object = {
					name : "luis"
				};
So if you want to access the value of the key name on object you'll get it like this 

				object.name; or object['name'];
The same for a hash in Ruby 

				hash = {
					name : "luis"
				}
				hash['luis']
The difference here is that hash.name is not a valid call, and it will give you an undefined method error, since hash doesn't have a method call name and "." in Ruby is use to call method within the object. In JavaScript a method is a property containing the function definition.
But where Ruby Hash is more powerful than an object in JavaScript is that it implements the module Enumerable, providing the hash with several build in methods that you would have to create yourself for a JavaScript object. For example loop methods.
Here is a compare table between Ruby Hash and JavaScript objects
Functionality rb Hash js Object Goes to
Data structure hash = { key: value} var object = { key : value} RB - JS
Accessing value hash[:key] or hash["key"] depending on the definition object["key"] or object.key JS
Creating a method hash = {:key -> { a*b }} or hash = {:key -> do a*b end} var object = { key : function(){ a*b } } RB
